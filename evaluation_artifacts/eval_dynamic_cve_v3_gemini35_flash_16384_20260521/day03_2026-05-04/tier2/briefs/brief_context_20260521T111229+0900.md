# Brief Context - Hanbit Regional Care Network (Day 3)

Hanbit Regional Care Network는 환자 포털, 예약 API, 내부 EHR 및 PACS 시스템을 운영하는 지역 거점 외래 진료 네트워크입니다.

## Organization And Asset Context
본 조직은 린(Lean) IT 및 보안 팀으로 운영되어 SOC 리소스가 제한적입니다. 주요 자산으로는 DMZ 내 환자 포털(`203.0.113.10`), VPN 게이트웨이(`203.0.113.20`), 예약 API(`203.0.113.30`)가 있으며, 내부망에는 Tomcat 기반의 Lab Results API(`10.60.20.30`)와 백업 NAS(`10.60.40.12`)가 존재합니다. 금일 자산 상태 파악 결과, Tomcat 관련 신규 취약점(CVE-2025-24813) 경보가 발령되어 예약 API 및 Lab API의 8080/8443 포트에 대한 비인가 접근 모니터링이 시급합니다.

## Current Risk Themes
- **Tomcat 취약점 악용**: `198.51.100.90` 등의 외부 위협 스캐너가 예약 API 및 내부 API의 8443 포트를 타겟팅하는 정황이 포착되었습니다.
- **백업 파괴 및 데이터 유출**: 일부 일반 워크스테이션에서 비인가 시간대에 백업 NAS로 SMB 연결을 시도하고, 백업 NAS가 외부 미확인 IP(`198.51.100.123`)로 다량의 데이터를 송신하는 랜섬웨어 전조가 관찰되었습니다.
- **기타 위협**: 외부 IP `198.51.100.77`에 의한 VPN 무차별 대입 공격 및 워크스테이션 내부의 IMDS 우회 접근 시도가 지속되고 있습니다.

## Over-Alerting Guardrails
- **정상 비즈니스 흐름 보호**: 주간 업무 시간(08:00-19:00 KST) 중 병원 워크스테이션 대역에서 발생하는 정상적인 API 통신 및 의료진의 PACS DICOM 접근은 경보에서 제외하십시오.
- **단순 자산 일치 경보 지양**: 단순히 대상 자산에 접근했다는 사실만으로 경보를 생성해서는 안 되며, 포트 오용, 비인가 소스 대역, 공격 성공 후 아웃바운드 행위가 결합된 경우에만 에스컬레이션하십시오.
